# InsForge 数据分析平台技术选型设计

> 创建日期：2026-06-29

## 项目概述

基于 InsForge 框架构建的企业数据分析平台，支持 API 数据采集、DuckDB 复杂计算、企业微信认证与推送，以及智能体 MCP 接入。

---

### 核心组件

| 组件 | 用途 |
|------|------|
| **Web 前端** | PC 端 + 企微 H5 移动端，报表查看 |
| **Edge Functions** | 业务逻辑：数据采集、预设查询 API、DuckDB 动态计算 |
| **PostgreSQL** | 业务数据 + 元数据 + DuckDB 内存表映射 |
| **Storage** | API 采集的原始数据存储（CSV/JSON/Parquet） |
| **Auth (企业微信扩展)** | 用户认证 + 组织架构权限映射 |
| **MCP Server** | 智能体接口：查询数据、获取元数据 |

## 二、数据流设计

### 2.1 数据采集流程

```
┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│  外部 API     │ ───→ │ Edge Function │ ───→ │   Storage    │
│  (业务系统)   │      │  (采集调度)   │      │  (原始数据)  │
└──────────────┘      └──────────────┘      └──────────────┘
                                                    │
                                                    ↓ 存储为文件
                                             ┌──────────────┐
                                             │ Parquet/CSV  │
                                             │ 按日期分区    │
                                             └──────────────┘
                                                    │
                                                    ↓ 元数据同步
                                             ┌──────────────┐
                                             │  PostgreSQL   │
                                             │  元数据表      │
                                             └──────────────┘
```

**采集策略**：
- 定时调度（Edge Function + cron）
- 文件命名：`/data/{source}/{date}/{type}.parquet`
- 元数据记录：数据源、采集时间、行数、schema

### 2.2 查询流程

```
┌─────────────────────────────────────────────────────────────┐
│                        查询请求                             │
│  ┌─────────────┐              ┌─────────────┐              │
│  │  Web 用户   │              │  智能体      │              │
│  └─────────────┘              └─────────────┘              │
└─────────────────────────────────────────────────────────────┘
         │                              │
         ↓ 预设 API                     ↓ MCP 接口
┌─────────────────────┐        ┌─────────────────────┐
│  Edge Function      │        │  MCP Server         │
│  查询预设数据        │        │  基础查询接口        │
└─────────────────────┘        └─────────────────────┘
         │                              │
         └──────────────┬───────────────┘
                        ↓
┌─────────────────────────────────────────────────────────────┐
│                      DuckDB 计算引擎                        │
│  1. 从 Storage 读取 Parquet 文件                            │
│  2. 与 PostgreSQL 业务表 JOIN                                │
│  3. 执行聚合/分析 SQL                                        │
│  4. 返回结果                                                 │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 数据模型（PostgreSQL）

```sql
-- 元数据表
CREATE TABLE data_sources (
  id UUID PRIMARY KEY,
  name VARCHAR(255),        -- 数据源名称
  api_endpoint VARCHAR(500), -- API 地址
  auth_type VARCHAR(50),    -- 认证方式
  schedule VARCHAR(100),    -- 调度规则
  created_at TIMESTAMP
);

CREATE TABLE data_files (
  id UUID PRIMARY KEY,
  source_id UUID REFERENCES data_sources(id),
  storage_path VARCHAR(500), -- Storage 文件路径
  file_format VARCHAR(20),   -- parquet/csv/json
  row_count INT,
  schema_json JSONB,         -- 字段定义
  created_at TIMESTAMP
);

-- 报表配置表
CREATE TABLE reports (
  id UUID PRIMARY KEY,
  name VARCHAR(255),
  description TEXT,
  query_template TEXT,       -- DuckDB 查询模板
  schedule VARCHAR(100),     -- 推送调度
  recipients JSONB,          -- 接收人列表
  created_at TIMESTAMP
);

-- 业务数据表示例
CREATE TABLE business_entities (
  id UUID PRIMARY KEY,
  name VARCHAR(255),
  owner_id UUID,             -- 关联企业微信用户
  created_at TIMESTAMP
);
```

## 三、智能体接入（MCP）

### MCP Tools 设计

| Tool 名称 | 功能 | 参数 |
|----------|------|------|
| `fetch-docs` | 获取平台文档和 schema | 无 |
| `list-reports` | 列出可访问报表 | `user_token` |
| `get-report` | 获取报表数据 | `report_name`, `user_token` |
| `query-table` | 查询预设数据表 | `table_name`, `filters`, `user_token` |

### MCP 接口示例

```typescript
// 1. 了解平台能力
await callMCP('fetch-docs');
// 返回：数据源列表、表结构、报表定义、使用说明

// 2. 获取可访问报表列表
await callMCP('list-reports', { user_token: 'xxx' });
// 返回：['销售日报', '运营周报', '财务月报']

// 3. 查询报表数据
await callMCP('get-report', {
  report_name: '销售日报',
  user_token: 'xxx'
});

// 4. 查询预设数据表
await callMCP('query-table', {
  table_name: 'employees',
  filters: { department: '研发部' },
  user_token: 'xxx'
});
```

### MCP Server 配置

```json
// openclaw MCP 配置
{
  "mcpServers": {
    "insforge": {
      "url": "https://data.yourcompany.com/mcp",
      "transport": "http"
    }
  }
}
```

## 六、部署架构

### 6.1 天翼云自托管架构

```
┌─────────────────────────────────────────────────────────────┐
│                    天翼云服务器                              │
│  ┌─────────────────────────────────────────────────────────┐│
│  │  Docker Compose                                          ││
│  │  ┌─────────────────────────────────────────────────────┐││
│  │  │  InsForge 容器组                                    │││
│  │  │  ├─ App Server (Web + API + MCP)   Port: 7130      │││
│  │  │  ├─ Auth Service                   Port: 7131      │││
│  │  │  ├─ Deno Runtime (Edge Functions)  Port: 7133      │││
│  │  │  └─ PostgREST API                  Port: 7140      │││
│  │  └─────────────────────────────────────────────────────┘││
│  │  ┌─────────────────────────────────────────────────────┐││
│  │  │  PostgreSQL + pgvector             Port: 5432      │││
│  │  └─────────────────────────────────────────────────────┘││
│  │  ┌─────────────────────────────────────────────────────┐││
│  │  │  MinIO / Storage                   Port: 9000      │││
│  │  └─────────────────────────────────────────────────────┘││
│  └─────────────────────────────────────────────────────────┘│
│  ┌─────────────────────────────────────────────────────────┐│
│  │  Nginx (反向代理 + SSL)                 Port: 80/443   ││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

### 6.2 服务器配置建议

| 配置项 | 推荐规格 | 说明 |
|--------|---------|------|
| **CPU** | 4 核 | DuckDB 计算需要 |
| **内存** | 16 GB | PostgreSQL + DuckDB |
| **存储** | 200 GB SSD | 数据量 < 100GB + 预留 |
| **带宽** | 10 Mbps | 企业内网访问足够 |
| **系统** | Ubuntu 22.04 / CentOS 8 | Docker 支持 |

### 6.3 Docker Compose 配置

```yaml
# docker-compose.yml
services:
  insforge:
    image: insforge/insforge:latest
    ports:
      - "7130:7130"   # App
      - "7131:7131"   # Auth
      - "7133:7133"   # Deno
    environment:
      - POSTGRES_HOST=postgres
      - STORAGE_HOST=minio
    depends_on:
      - postgres
      - minio

  postgres:
    image: postgres:15
    ports:
      - "5432:5432"
    environment:
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data

  minio:
    image: minio/minio
    ports:
      - "9000:9000"
      - "9001:9001"
    environment:
      - MINIO_ROOT_USER=${MINIO_USER}
      - MINIO_ROOT_PASSWORD=${MINIO_PASSWORD}
    volumes:
      - minio_data:/data
    command: server /data --console-address ":9001"

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./ssl:/etc/nginx/ssl

volumes:
  postgres_data:
  minio_data:
```

## 七、技术栈总结

### 后端

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| **平台框架** | InsForge（自托管） | 一体化后端平台 |
| **语言** | TypeScript | Edge Functions 开发 |
| **数据库** | PostgreSQL 15 + pgvector | 业务数据 + 向量扩展 |
| **计算引擎** | DuckDB | 复杂数据分析 |
| **对象存储** | MinIO | S3 兼容，存储采集数据 |
| **认证** | InsForge Auth + 企业微信 OAuth | 组织架构集成 |
| **智能体接口** | MCP Server | openclaw 接入 |

### 前端

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| **框架** | Next.js 14+ | PC + 移动端 H5 |
| **UI 组件** | shadcn/ui | 可定制，响应式 |
| **图表** | ECharts | 移动端适配好 |
| **表格** | TanStack Table | 数据展示 |
| **企微 SDK** | @wecom/jssdk | 企微 H5 集成 |

### 运维

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| **容器** | Docker + Docker Compose | 服务编排 |
| **反向代理** | Nginx | SSL + 路由 |
| **云服务商** | 天翼云 | 私有化部署 |
| **SSL 证书** | 天翼云 SSL / Let's Encrypt | HTTPS |
